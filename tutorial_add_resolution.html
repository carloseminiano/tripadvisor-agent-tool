📘 Step-by-Step Tutorial: Adding a New Resolution

🔹 Step 1: Understand the Flow

Each resolution is triggered based on 5 dropdowns:

  1. Requestor
  2. Category
  3. Concern
  4. Sub-Concern
  5. Condition 2 (Specific state)

These combine to create a resolution key:

requestor_category_concern_subconcern_condition2

🔹 Step 2: Update Dropdown Options

In your HTML, find this section in the <script>:

A. Add to categoryMap:

const categoryMap = {
  "owner": ["listing", "product", "booking"],
  "member": ["member_account", "booking"]
};

B. Add to concernMap:

const concernMap = {
  "product": ["billing", "cancel"]
};

C. Add to subConcernMap:

const subConcernMap = {
  "billing": ["refund", "payment_issue"]
};

D. Add to condition2Map:

const condition2Map = {
  "refund": ["credit_card", "paypal"]
};

🔹 Step 3: Add the Resolution

Scroll to the resolutions section:

const resolutions = {
  "owner_product_billing_refund_credit_card": `
    <strong>Resolution:</strong><br><br>
    💳 Inform the customer that credit card refunds take 3–5 business days.<br>
    Verify charge ID and status in account center.<br><br>

    📩 <b>Email Template:</b> <span id="emailTemplate2">Product_Billing_CreditCard_Refund</span>
    <button onclick="copyText('emailTemplate2')">📋 Copy</button><br>

    📚 <b>KB Article:</b> <span id="kbLink2">Billing & Refund Policy</span>
    <button onclick="copyText('kbLink2')">📋 Copy</button><br>

    🗂 <b>Related Cases:</b> 1100101, 1100456
  `
};

💡 Tips

✅ Keep your keys consistent.
Use lowercase and underscore `_` between each word.

Example: owner_product_billing_refund_credit_card
📝 Note:
You do not need to modify any HTML layout—just work inside the JavaScript area.
